# Java'da `?`, `Object.class` va `T` o‘rtasidagi taqqoslash

## 1. `?` (Wildcard)
- **Ta’rifi:** Wildcard (`?`) Java'da generics bilan ishlaganda **noma’lum tip**ni ifodalaydi. U uch shaklda bo‘ladi:
  - **Unbounded Wildcard (`?`):** Har qanday tip.
  - **Upper Bounded Wildcard (`? extends T`):** `T` yoki uning voris tiplari.
  - **Lower Bounded Wildcard (`? super T`):** `T` yoki uning ota-classlari.
- **Maqsadi:** Genericsda moslashuvchanlikni ta’minlash, aniq tip talab qilmasdan umumiy tiplar bilan ishlash.
- **Ishlatilishi:** Metodlarda parametr sifatida, kolleksiyalar bilan moslashuvchanlik uchun.
- **Misol:**
  ```java
  public void printList(List<?> list) {
      System.out.println(list);
  }
  ```
  Bu metod `List<String>`, `List<Integer>` yoki boshqa tiplarni qabul qiladi.
- **Cheklovlari:** 
  - `?` bilan yozish (`add`) cheklangan (faqat `null` qo‘shish mumkin).
  - `? extends T` faqat o‘qish, `? super T` asosan yozish uchun.

## 2. `Object.class`
- **Ta’rifi:** `Object.class` Java'da `Object` classining class ob’ekti bo‘lib, barcha classlarning ota-classi hisoblanadi.
- **Maqsadi:** Umumiy tip sifatida ishlatish yoki reflection API’da class haqida ma’lumot olish.
- **Ishlatilishi:** Generics ishlatilmaydigan kodlarda yoki umumiy tip kerak bo‘lganda.
- **Misol:**
  ```java
  public void printList(List<Object> list) {
      System.out.println(list);
  }
  List<Object> objList = new ArrayList<>();
  objList.add("Salom");
  objList.add(123);
  printList(objList); // [Salom, 123]
  ```
  Ammo `List<String>` ni `List<Object>` sifatida qabul qilib bo‘lmaydi.
- **Cheklovlari:** Tip xavfsizligi yo‘q, qo‘lda casting talab qilinishi mumkin.

## 3. `T` (Generic Tip Parametri)
- **Ta’rifi:** `T` genericsda umumiy tip parametri bo‘lib, class yoki metodda aniq tipni ifodalaydi.
- **Maqsadi:** Tip xavfsizligini ta’minlash va kodni qayta ishlatiladigan qilish.
- **Ishlatilishi:** class yoki metodda aniq tip bilan ishlash uchun.
- **Misol:**
  ```java
  public class Box<T> {
      private T item;
      public void setItem(T item) { this.item = item; }
      public T getItem() { return item; }
  }
  Box<String> stringBox = new Box<>();
  stringBox.setItem("Salom");
  System.out.println(stringBox.getItem()); // "Salom"
  ```
- **Cheklovlari:** Faqat bitta aniq tip bilan ishlaydi.

## 4. Taqqoslash

| **Xususiyat**                | **`?` (Wildcard)**                              | **`Object.class`**                           | **`T` (Generic Tip)**                      |
|------------------------------|-----------------------------------------------|---------------------------------------------|--------------------------------------------|
| **Ta’rif**                   | Noma’lum tip, moslashuvchan.                  | Barcha classlarning ota-classi.                | Aniq umumiy tip parametri.                 |
| **Tip xavfsizligi**          | Qisman (o‘qish/yozish cheklovlari bor).       | Yo‘q, casting talab qilinadi.               | Ha, kompilyatsiya vaqtida xavfsiz.         |
| **Moslashuvchanlik**         | Yuqori (`extends`/`super` bilan).              | Yuqori, lekin xavfsiz emas.                 | Cheklangan, faqat bitta tip.               |
| **Ishlatilishi**             | Metodlarda, kolleksiyalar bilan.              | Umumiy tip yoki reflection uchun.           | class/metodlarda aniq tip uchun.            |
| **O‘qish/Yozish**            | Cheklangan (`extends` o‘qish, `super` yozish). | Cheklov yo‘q, lekin xavfsiz emas.           | To‘liq, lekin faqat bitta tip uchun.       |
| **Misol**                    | `List<? extends Number>`                      | `List<Object>`                              | `Box<String>`                              |

## 5. Amaliy misollar

### Misol 1: Wildcard
```java
public void printWildcard(List<? extends Number> list) {
    for (Number num : list) {
        System.out.println(num);
    }
    // list.add(10); // Xato, yozish mumkin emas
}
```

### Misol 2: Object
```java
public void printObject(List<Object> list) {
    list.add("Test"); // Xavfsiz emas
    System.out.println(list);
}
```

### Misol 3: Generic T
```java
public <T extends Number> void printGeneric(List<T> list) {
    for (T num : list) {
        System.out.println(num);
    }
    list.add((T) Integer.valueOf(10)); // Faqat T tipidagi element
}
```

### Misol 4: Taqqoslash
```java
public static void main(String[] args) {
    List<Integer> intList = Arrays.asList(1, 2, 3);
    printWildcard(intList); // Ishlaydi
    // printObject(intList); // Xato, List<Integer> != List<Object>
    printGeneric(intList); // Ishlaydi
}
```

## 6. Xulosa
- **`?` (Wildcard):** Moslashuvchanlik uchun, metodlarda umumiy tiplar bilan ishlashda ideal.
- **`Object.class`:** Umumiy, lekin tip xavfsizligi yo‘q, eski kodlar yoki reflection uchun.
- **`T` (Generic Tip):** Aniq tip xavfsizligini ta’minlaydi, lekin faqat bitta tip bilan ishlaydi.
